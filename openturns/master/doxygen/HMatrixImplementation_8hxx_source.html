37 // In order to create an H-matrix, one has first to generate an interface with
38 // HMatrixFactory::build, then compute its coefficients, and after that, it is
39 // possible to factorize it, solve linear systems, etc. As H-matrices are
40 // designed to solve very large systems, most operations are performed inplace,
41 // which is an unusual pattern in OpenTURNS.
42 
43 // There are two ways to compute matrix coefficients:
44 // 1. The simplest solution is to provide a method which computes (i,j) coefficient.
45 // This is what HMatrixRealAssemblyFunction is designed for, and such an object
46 // is then passed to HMatrixImplementation::simpleAssemble. But most of the time,
47 // this involves heavy computations which could be reused when computing values
48 // for different coefficients.
49 // 2. HMat provides a 2nd interface to compute tiled blocks. But it is cumbersome,
50 // so we defined a more intuitive interface which should be suited for our needs.
51 // When dimension is greater than 1, HMatrixFactory::build duplicates input vertices
52 // to have local d x d matrices, and HMatrixTensorRealAssemblyFunction is an interface
53 // to compute all coefficients of this local matrix, which had already been allocated.
54 // It will be automatically copied at the right place in HMatrixImplementation.
